### 핫 리로딩(hot reloading)

앱을 처음부터 다시 시작하지 않고 새로운 코드 변경에따른 코드 변경사항만 표시하며 변경된 코드에만 적용

- 기존 방식 → 저장 → 리로딩 → 상태 초기화
- 핫 리로딩 → 저장 → **변경된 UI만 갱신** → **상태 유지**


##### 장점
- 수정 뒤에도 애플리케이션의 상태가 사라지지 않음.
###### gpt 사용
- 즉시 코드 반영

- 빠르게 UI/UX테스트 가능
	- 상태 초기화가 없어서 빠르게 할 수 있음.

- 개발 효율성이 향상될 수 있음
	- 반복적인 새로고침 불필요
	- 코드 수정 -> 바로 적용 -> 오류 발견 -> 바로 수정

##### 단점(GPT 말이 좀 많이 많음)
- 버그가 계속 유지될 수 있음
	- 상태 초기화 x -> 이미 질못된 상태로 앱이 돌아가고 있다면 문제가 계속될 수 있음.
- 코드 변경이 상태 구조를 바꾸면 **충돌 가능**

- 프로덕션과 다르게 작동할 가능성이 있음
	- 프로덕션에서는 상태가 **매번 새로 시작**됨

- 디버깅 혼란이 생길 수 있음
	- 현재 코드와 화면 상태가 불일치하는 순간이 생길 수 있음.


### 시간 여행 디버깅
이전의 **특정 상태로 돌아갈 수** 있게 해주는 것

##### 장점
- 이전 버전으로 언제든지 돌아갈 수 있다.
- 문제 파악이 용이하다.

##### 단점(GPT)
- 메모리 사용량 증가
- 성능 저하
- 비동기 처리 어려움
- 복잡한 상태 구조 문제
