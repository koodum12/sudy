## 1. 생성형 AI

### 사전적 정의

- 방대한 데이터를 고도화된 알고리즘으로 학습한 생성 모델이 사람이 필요로 하는 지식을 추론하여 생성
    
- AI는 본질적으로 내용을 **이해하지 않음**
    
    - 하지만 트랜스포머 등 모델은 데이터 속 관계를 추적하여 맥락을 파악
        

### Code

- GitHub Copilot, DeepCode, OpenAI Codex 등 코드 생성 프로그램
    
- 간단한 텍스트 명령만으로 한 페이지 코드 생성 가능
    
- 사람이 작성한 코드를 더 우수한 코드로 개선 가능
    
- (이런 프로그램이 있어도 개발자는 지속적으로 공부해야 함)
    

### 영상 생성이 주는 의미

- 기존 동영상 = 현실 콘텐츠, 생성 AI가 패러다임 변화
    
- 이질적인 존재들이 섞이며 새로운 개념 탄생
    
- 비현실적 현실 조작 우려 심화
    

### 생성 속도

- MIT 연구: 작가와 데이터 분석가 444명 대상 보고서 작성 실험
    
- GPT 활용 집단, 보고서 작성 시간 50% 단축
    
- 내용상 더 높은 평가 받음
    

### 생산 속도

#### 초보자

- AI를 활용해 빠르게 적응 가능
    

#### 전문가

- 업무 속도 향상
    
- 10명이 해야 할 일을 2~3명이 수행 가능
    
- 1명이 여러 AI를 활용하여 기업 단위의 일 처리 가능
    

### 활용 영역

- 투자 대비 창출 가치 임팩트
    
    - 고객관리
        
    - 제품 개발
        
    - 광고 제작
        
    - 가상 모델 생성
        
    - 건축 디자인

## Semantic Kernel

### 정의

- AI 오케스트레이션을 위한 도구
    
- 여러 LLM 모델을 활용하여 다양한 역할을 수행
    

### 장점

- 기업용 앱 개발 친화적
    
    - 기존 앱 개발 경험 유지
        
    - 레거시 앱과 통합 가능
        
    - 스케일링 및 보안 지원
        

### 플러그인

- 기존 API와 연결 가능
    
- 프롬프트 플러그인
    
- 네이티브 코드 플러그인
    

### 멀티 에이전트

- 다양한 페르소나 설정 가능
    
- 여러 에이전트 조율 가능
    

### LangChain vs Semantic Kernel

- **Semantic Kernel**은 다양한 기술 스택과 호환 가능

## RAG (Retrieval-Augmented Generation)

### 개념

- 검색 증강 생성 기법
    
- 최신 데이터 적용 가능
    
- 보안 및 기밀 유지 지원
    

### 비정형 텍스트 데이터 처리

- 이미지, 동영상, 코드 등 비정형 데이터 활용 가능
    
- 이를 위해 벡터 데이터베이스 사용 필요
    

### 주요 Vector DB

- Azure Cosmos DB
    
- Redis
    
- MongoDB

## OpenTelemetry

### 정의

- 오픈소스 분산 추적 및 모니터링 프로토콜
    
- 애플리케이션의 성능 및 상태를 모니터링하는 표준
    

### 기능

- 로그, 메트릭, 추적 정보를 수집 및 분석
    
- AI 애플리케이션의 성능을 최적화하고 오류를 신속하게 파악 가능
    

### 주요 활용

- 클라우드 네이티브 애플리케이션 모니터링
    
- 분산 시스템의 성능 분석
    
- DevOps 및 AI 기반 서비스 품질 관리
